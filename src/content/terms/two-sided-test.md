title: 両側検定

exam_scope:

level: standard

category: hypothesis_testing
short_definition: 対立仮説で基準との差を上下どちらの方向でも認め、分布の両端で判定する仮説検定です。
definition: 両側検定とは、対立仮説を「基準より大きいか小さいかに関係なく、とにかく異なる」と置く仮説検定です。母平均なら $H_1:\mu\ne\mu_0$ の形になります。このとき、有意水準 $\alpha$ は分布の左右の尾に分けて置かれます。したがって、片側検定よりも一方向あたりの棄却域は狭くなります。増加も減少もどちらも重要なときに使うのが基本です。
intuition:
  - 両側検定は、「高くなったか低くなったか」ではなく「違うか」を問う検定です。
  - 基準から上にも下にも外れる可能性を同じ重みで見ます。
  - 片側検定より慎重なのは、有意水準を両端に分けて使うからです。
  - 観測値の符号で向きは分かりますが、検定の目的自体は「どちらでも違えば重要」です。
  - 上下どちらのずれも困る品質管理では、両側検定が自然です。
visual_explanation:
  - 帰無仮説の分布の左右両端を棄却域として塗る図を思い浮かべると理解しやすくなります。
  - 観測値が中央から大きく離れていても、右でも左でも同じ基準で判断します。
  - 有意水準 5% なら、標準的には左右 2.5% ずつに分けます。
  - p値も、観測値と同じ大きさだけ反対側に離れた領域を合わせて考えます。
where_it_appears:
  - 平均値が基準より高すぎても低すぎても困る品質管理
  - 新旧の方法に差があるかを、方向を決めずに調べる場面
  - 平均や比率が「変化したか」を問う統計検定2級の標準問題
  - 片側検定との臨界値や p値の違いを確認する場面
practical_examples:
  - title: 平均内容量が 500g と違うか
    description: 商品が重すぎても軽すぎても問題なら、対立仮説は $H_1:\mu\ne500$ です。標本から $Z=1.8$ が出ても、$\alpha=0.05$ の両側検定では臨界値約 1.96 に届かないので棄却しません。
  - title: 教材で成績が変わったか
    description: 新教材で点数が上がる可能性も下がる可能性もあり、どちらも重要なら両側検定を使います。単に「上がったか」だけを問う片側検定とは違います。
  - title: 上下どちらのずれも異常
    description: 製品の長さが基準より長すぎても短すぎても不良になる場合、両側検定で両端を監視する考え方が自然です。
exam_points:
  - 両側検定は、対立仮説が $\ne$ を使う形になるときに使います。
  - 有意水準 $\alpha$ は左右の尾に分けて置かれます。
  - 同じ $\alpha$ なら、片側検定より一方向で棄却しにくくなります。
  - p値は、観測値と同じくらい極端な両側の確率として読むことが多いです。
  - 方向を決めずに差を見たいときは、片側検定ではなく両側検定を使います。
formulas:
  - name: 両側検定の棄却域
    latex: 'H_1:\mu\ne\mu_0,\quad R=\{z \mid |z|>z_{1-\alpha/2}\}'
    description: 両側検定では、標準化した統計量 $z$ が正負どちら側でも十分大きければ棄却します。有意水準は両端に $\alpha/2$ ずつ分かれます。
    conditions:
      - '$R$ は棄却域です。'
      - '$z_{1-\alpha/2}$ は標準正規分布の累積確率 $1-\alpha/2$ に対応する臨界値です。'
      - z 検定での代表的な形です。
  - name: 両側 p値
    latex: 'p=P_{H_0}(|T|\ge |t_{\mathrm{obs}}|)'
    description: 観測された検定統計量 $t_{\mathrm{obs}}$ と同じくらい極端な値が、正負どちら側にも出る確率を合わせたものが両側 p値です。
    conditions:
      - '$T$ は帰無仮説のもとでの検定統計量です。'
      - 対称な分布をもつ典型的な検定でよく使う書き方です。
rigorous_explanation:
  - 両側検定では、帰無仮説からのずれの向きを固定しません。そのため、棄却域は通常、分布の左右両端に分かれます。
  - 同じ有意水準なら、片側検定より各尾に割り当てられる確率が小さくなるので、臨界値は中心から遠くなります。これが、片側検定より棄却しにくい理由です。
  - 両側検定は「差があるか」を問うのであって、「どちら向きに差があるか」を捨てるわけではありません。棄却したあとに、観測統計量の符号で方向は読み取れます。
  - 片側にしてよいか迷うときは、両方向を問題にする両側検定を選ぶほうが安全です。
proof:
  summary: 両側検定では、有意水準を左右の尾に分けるため、片側検定より臨界値が外側に移ります。
  outline_steps:
    - '全体の有意水準を $\alpha=0.05$ とします。'
    - '両側検定では、上側 2.5% と下側 2.5% の合計 5% を棄却域にします。'
    - '標準正規分布では、上側 2.5% に対応する臨界値は $z_{0.975}\approx1.96$ です。'
    - 'したがって、$|z|>1.96$ のときだけ棄却になり、片側検定の 1.645 より厳しい基準になります。'
common_mistakes:
  - 「違いがあるか」という問いなのに片側検定を使うこと
  - 両側検定でも片側の臨界値 $z_{1-\alpha}$ を使ってしまうこと
  - 両側 p値と片側 p値を同じ意味で比較すること
  - データを見てから両側検定を片側検定へ変えること

`one-sided-test` と比べると、両側検定は「どちら向きの差も見逃したくない」場面に向いています。だからこそ、有意水準を左右に分けるぶん、一方向だけを見る片側検定より慎重な基準になります。

この違いは `alternative-hypothesis` の形に直結します。問題文に「増えたか」ではなく「違いがあるか」と書かれているなら、まず両側検定を疑うと整理しやすいです。
