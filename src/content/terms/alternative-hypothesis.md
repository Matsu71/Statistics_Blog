title: 対立仮説

category: hypothesis_testing
short_definition: 仮説検定で、帰無仮説ではない方向の差や効果を表す仮説です。
definition: 対立仮説とは、帰無仮説 $H_0$ に対して「差がある」「大きくなっている」「小さくなっている」といった、検出したい変化の方向を表す仮説です。記号では $H_1$ と書きます。対立仮説は、単に帰無仮説の反対というだけでなく、どの方向の差を問題にしているかを決める役割を持ちます。そのため、片側検定にするか両側検定にするかは、対立仮説の形から決まります。対立仮説をデータを見たあとで都合よく選んではいけません。
intuition:
  - 対立仮説は、「本当に見つけたい変化は何か」を言葉と記号で表したものです。
  - 平均が高くなったかを知りたいのか、単に違いがあるかを知りたいのかで、対立仮説は変わります。
  - 検定の向きは、観測後に決めるのではなく、検定前に決めておきます。
  - 帰無仮説が基準役なら、対立仮説は「どちらに外れたら意味があるか」を示す役です。
  - 対立仮説は研究の関心を表しますが、棄却したからといって無条件に証明されたわけではありません。
visual_explanation:
  - 右側だけを見る対立仮説なら、分布の右端に注目します。
  - 左側だけを見る対立仮説なら、分布の左端に注目します。
  - 「違いがある」とだけ言う対立仮説なら、分布の両端を見ることになります。
  - 同じ観測値でも、どの対立仮説を置くかで p値や棄却域の形が変わります。
where_it_appears:
  - 新しい施策で平均点が上がったかを調べる場面
  - 不良率が基準より下がったかを確認する場面
  - ある測定値が基準から上にも下にもずれる可能性を考える場面
  - 片側検定と両側検定のどちらを使うかを決める場面
practical_examples:
  - title: 教材の効果が上向きかを調べる
    description: 新教材で平均点が上がったかだけを問題にするなら、対立仮説は $H_1:\mu>70$ のように右片側になります。平均点が下がる可能性を同じ重みで調べる検定とは別物です。
  - title: 製品の平均重量が基準と違うかを見る
    description: 平均重量が重すぎても軽すぎても困るなら、対立仮説は $H_1:\mu\ne500$ です。この場合は両側検定で両端を見ます。
  - title: 不良率が改善したかを確認する
    description: 工程改善で不良率が下がったかを見たいなら、対立仮説は $H_1:p<p_0$ のように左片側になります。単に「違う」ではなく、下がる方向を指定しています。
exam_points:
  - 対立仮説 $H_1$ は、どの方向の差や効果を見たいかを表します。
  - 片側検定か両側検定かは、対立仮説の形から決まります。
  - 対立仮説は標本を見る前に決めます。
  - 等号を含む形は通常、対立仮説ではなく帰無仮説に置かれます。
  - 帰無仮説を棄却したときは、対立仮説を支持する証拠が強まったと読みます。
formulas:
  - name: 右片側の対立仮説
    latex: 'H_1:\mu>\mu_0'
    description: 母平均が基準値 $\mu_0$ より大きい方向だけを問題にする形です。右片側検定につながります。
    conditions:
      - '$\mu$ は母平均です。'
      - '$\mu_0$ は比較の基準値です。'
      - 上方向の差だけを意味ある変化として扱うときに使います。
  - name: 左片側の対立仮説
    latex: 'H_1:\mu<\mu_0'
    description: 母平均が基準値より小さい方向だけを問題にする形です。左片側検定につながります。
    conditions:
      - 下方向の差だけを見たいときに使います。
      - 基準値から上にずれる可能性を同じ重みでは扱いません。
  - name: 両側の対立仮説
    latex: 'H_1:\mu\ne\mu_0'
    description: 母平均が基準値と異なるかを、上方向と下方向の両方で見る形です。両側検定につながります。
    conditions:
      - 上下どちらのずれも問題になるときに使います。
      - 棄却域は通常、分布の両端に分かれます。
rigorous_explanation:
  - 対立仮説は、検定の向きと棄却域の形を決めます。同じ検定統計量でも、$H_1:\mu>\mu_0$ と $H_1:\mu\ne\mu_0$ では p値の定義が変わります。
  - 等号が帰無仮説に置かれやすいのは、その値のもとで分布を具体的に定められるからです。そのため、対立仮説は通常、不等号や $\ne$ を使う形になります。
  - データを見たあとで片側と両側を選び直すと、あらかじめ定めた有意水準の意味が崩れます。検定の設計は観測前に決める必要があります。
  - 対立仮説が複合的な場合、第二種の過誤や検出力は、どの真の値を考えるかによって変わります。つまり、$H_1$ は「差がある」という一言以上の役割を持ちます。
proof:
  summary: 対立仮説を決めると、どちら側を極端とみなすかが定まり、片側検定か両側検定かも自動的に決まります。
  outline_steps:
    - 'まず、研究の関心が「大きい方向」「小さい方向」「どちらでも違う方向」のどれかを整理します。'
    - 'その関心を $H_1:\mu>\mu_0$、$H_1:\mu<\mu_0$、$H_1:\mu\ne\mu_0$ のどれかで表します。'
    - '右側だけを見るなら右片側、左側だけなら左片側、両方なら両側の棄却域になります。'
    - 'したがって、対立仮説は「何を見つけたいか」だけでなく、「どう判定するか」まで決める要素だと分かります。'
common_mistakes:
  - データを見たあとで、有利そうな向きの対立仮説を選ぶこと
  - 「違いがある」と「大きい」を同じものとして扱うこと
  - 等号を含む仮説を対立仮説に置くこと
  - 帰無仮説を棄却したら、対立仮説が完全に証明されたと思い込むこと

`null-hypothesis` と並べて読むと、対立仮説は「帰無仮説の逆」というだけでなく、「どの方向の差を見に行くか」を決める役だと分かります。特に、片側か両側かの違いは対立仮説の書き方にそのまま表れます。

`one-sided-test` と `two-sided-test` は、対立仮説を実際の判定規則に直した姿です。問題文が「増えたか」なのか「違うか」なのかを読み分けると、どちらを使うべきか判断しやすくなります。
