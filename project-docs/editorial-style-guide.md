# Editorial Style Guide

## 1. 目的と対象読者

このガイドは、統計検定2級向けの「1用語1ページ」学習用語集を、AI を使っても表記ゆれしにくく、初学者にも読みやすい形で量産するための編集基準である。

- 主対象は、文系の初学者を含む統計学習者
- 入口はやさしく、内容は浅くしすぎない
- 直感、定義、試験での重要点、数式、厳密な説明の順に理解を深める
- 厳密さより分かりやすさを優先するのではなく、両立を目指す

## 2. 用語ページの標準構成

各用語ページは、原則として次の順に構成する。frontmatter の構造化項目と対応させ、本文 Markdown へ逃がしすぎない。

1. `この用語は何か`
   - `short_definition`
2. `まずはイメージで理解する`
   - `intuition`
3. `視覚的・直感的な説明`
   - `visual_explanation`
4. `定義`
   - `definition`
5. `どのような場面で現れるか`
   - `where_it_appears`
6. `現実の例・実務の例`
   - `practical_examples`
7. `試験で重要なポイント`
   - `exam_points`
8. `数式と記号`
   - `formulas`
9. `厳密な説明`
   - `rigorous_explanation`
10. `証明`
    - `proof`
11. `間違えやすい点`
    - `common_mistakes`
12. `関連用語`
    - `related_terms`
13. `参考文献`
    - `references`

補足本文は Markdown に書いてよいが、構造化すべき内容を本文だけに置かない。

## 3. 文体と説明トーン

- 文体は常に `です・ます調` に統一する
- 最初の 2 段落以内で、短い定義とその役割を示す
- 1 文 1 論点を原則にする
- 読者を置いていく煽り口調や断定口調を避ける
- 「何を知るための概念か」を先に伝える
- 直感説明を先に置き、その後で定義・数式・厳密説明へ進む

## 4. 統計用語の統一方針

### 主表記

- `平均`
- `分散`
- `標準偏差`
- `不偏分散`
- `母集団`
- `標本`
- `母平均`
- `母分散`
- `標本平均`
- `確率変数`
- `確率分布`
- `二項分布`
- `正規分布`
- `推定`
- `点推定`
- `区間推定`
- `仮説検定`
- `p値`
- `有意水準`
- `相関係数`
- `回帰直線`

### 補足扱いにする表記

- `算術平均`: 初出のみ補足可
- `危険率`: `有意水準` の補足としてのみ使用可

### 禁止・非推奨

- `P値`: `p値` に統一する
- `相関があるので因果関係がある`: 不正確なので禁止
- `標本分散` と `不偏分散` を区別せずに使うこと

## 5. 数式と記号の書き方

- 短い記号は `$x_i$` のような inline math、主要な式は `formulas[].latex` または Markdown の display math にする
- `formulas[].latex` は KaTeX でレンダリングされるため、正規の LaTeX を書く
- `√`、`Σ`、`1 / n` のようなプレーン表記ではなく、`\sqrt{}`、`\sum`、`\frac{}{}` を優先する
- 数式を出したら、その直後に日本語で意味を説明する
- display 数式の直前には、何を見る式なのかを 1 文で置く
- 記号は初出で必ず説明する
- 母数と標本統計量の記号を混同しない
- 同じページで記号をぶらさない
- 近似は `≈`、定義や恒等式は `=` を使う
- 長い式は無理に本文へ入れず、display 数式として独立させる
- 証明や厳密説明の中でも、短い式は inline math にし、式だけで段落を始めない

### 記号の基本方針

- 母平均: `μ`
- 母分散: `σ^2`
- 標本平均: `\bar{x}`
- 不偏分散: `s^2`
- 確率: `P(A)`, `P(X=x)`

## 6. 初学者向け説明の原則

- まず概念の役割を言う
- 次に小さな数字や身近な例で直感を作る
- その後に定義と式へ進む
- 最後に混同しやすい概念を整理する
- 直感説明は厳密な定義と矛盾させない
- 実務例を入れる場合も、試験範囲との接点を失わない

## 7. 避ける表現と禁止事項

- `なんとなくこうなる`
- `とにかく覚えましょう`
- `超重要`
- `めちゃくちゃ簡単`
- 定義前に専門用語を連打する説明
- `p値が小さいほど帰無仮説が正しくない確率が高い`
- 記号の未定義使用
- 初学者向け本文に証明を混ぜ込み、難所の境目を消すこと

## 8. AI 生成時のチェックリスト

- タイトルと `short_definition` が一致しているか
- `definition` と `short_definition` が矛盾していないか
- `where_it_appears` と `practical_examples` が重複していないか
- 母集団と標本の記号が混線していないか
- `p値` と `有意水準` を取り違えていないか
- `不偏分散` と `標本分散` を曖昧にしていないか
- すべての数式に記号説明が付いているか
- `common_mistakes` がその用語固有の注意になっているか
- `related_terms` が実在の slug と対応しているか
- 見出し順が標準構成から崩れていないか
- 参考文献を捏造していないか

## 9. 運用メモ

- `references` は空でもよい。未確認の参考文献を AI が作らない
- `proof` は必須ではない。自然な場合だけ入れる
- `区間推定` と `信頼区間` の主従関係は今後さらに明文化する
- `標本分散` と `不偏分散` のページ分割方針は queue と合わせて管理する
